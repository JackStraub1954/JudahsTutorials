Digression: equals and hashCode for Maps and Sets: In the above code we test corresponding LinePropertySets for equality by comparing the maps that contain them:
     this.linePropertySetMap.equals( that.linePropertySetMap );
This leads us to a discussion of how Maps (and subsequently Sets) are compared for equality.

The documentation for Map<K,V> says that two maps are equal if "...the two maps represent the same mappings. More formally, two maps m1 and m2 represent the same mappings if m1.entrySet().equals(m2.entrySet())." Which leads to a discussion of Sets.

The documentation for Set<E> says the equals "[returns] true if the specified object is also a set, the two sets have the same size, and every member of the specified set is contained in this set." Then there's a qualification: "Note: Great care must be exercised if mutable objects are used as set elements. The behavior of a set is not specified if the value of an object is changed in a manner that affects equals comparisons while the object is an element in the set."
