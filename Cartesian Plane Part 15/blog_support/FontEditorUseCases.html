Font Editor Use Cases

GUI Component Description

font-editor
An implementation of the FontEditor class; a collection of components as described below.
font-name-list
A list of font names including, at a minimum:
  • Dialog
  • DialogInput
  • Monospaced
  • SansSerif
  • Serif
bold-toggle
A check box which indicates whether or not a font should be bold.
italic-toggle
A check box which indicates whether or not a font should be italic.
size-editor
A component used to edit the font size.
size-editor
A component used to edit the font size.
color-button
A button that posts a color selector, as described in the ColorEditor specification.
color-editor
A component used to edit a color, as described in the ColorEditor specification.
feedback-window
A JLabel that displays text that reflects the selected font properties and text color.

Properties and Initial Values

font-name
The name of the font; synchronized with the feedback window and the font-name-list.
Initial value: Dialog
bold
The font weight; synchronized with the feedback window and the bold-toggle.
Initial value: false
italic
The font slope; synchronized with the feedback window and the italic-toggle.
Initial value: false
font-size
The font slope; synchronized with the feedback window and the size-editor.
Initial value: 10
text-color
The text color; synchronized with the feedback window and the color-editor.
Initial value: blue (0xFF)

Use Cases

construction
The property accessors return the correct initial values; the components reflect those same initial values.
font-name-selection
Instantiate the FontEditor. For each name in the font-names list: select the name; verify that getName() returns the correct name. Verify that:
  1. Accessors for for bold, italic, font-size and text-color return the initial values for those properties.
  2. getFont() returns a font that matches the selected properties.
  3. the font in the feedback window matches the value returned by getFont().
